# Admin CMS Design Spec

## Mission
Build a comprehensive Content Management System within the /admin dashboard featuring centralized content control, AI-powered blog creation, calendar-driven scheduling, automated content pipeline, newsletter composition, and official correspondence management.

## Architecture

### Approach
Modular admin components following the existing SPA hash-routing pattern. Break the 1,695-line monolith into separate components. Add new Prisma models, REST API routes, ZAI SDK integration for AI content generation, and MDXEditor for rich text editing.

### Admin Views (10)
1. **Dashboard** — Overview stats, upcoming calendar events, quick actions
2. **Blog Posts** — CRUD with MDXEditor rich text, SEO metadata fields
3. **AI Blog Engine** — Generate articles from topic/category, review, edit, publish
4. **Content Calendar** — Monthly/weekly calendar showing all scheduled content
5. **Auto-Scheduler** — Configure bi-weekly AI generation, populate rest of year
6. **Newsletters** — Compose, template, save drafts with variable injection
7. **Correspondence** — Letters/memos/notices with template builder
8. **Templates** — Shared template manager for newsletter + correspondence
9. **Partners** — Existing partner management (migrated from monolith)
10. **Settings** — Existing site settings (migrated from monolith)

## Database Changes

### BlogPost (existing — add fields)
- `scheduledAt` DateTime? — auto-schedule date
- `autoGenerated` Boolean @default(false) — AI-generated flag
- `aiPrompt` String? — prompt used for generation
- `metaTitle` String? — SEO meta title
- `metaDescription` String? — SEO meta description
- `tags` String? — comma-separated tags

### Newsletter (new)
- id String @id @default(cuid())
- title String
- subject String
- content String — rich text HTML/MDX
- templateId String? — FK to ContentTemplate
- status String @default("draft") — draft | approved | archived
- createdAt DateTime @default(now())
- updatedAt DateTime @updatedAt

### Correspondence (new)
- id String @id @default(cuid())
- title String
- type String — letter | memo | notice | invitation
- content String — rich text HTML/MDX
- templateId String? — FK to ContentTemplate
- recipientName String?
- recipientOrganization String?
- status String @default("draft") — draft | approved | archived
- createdAt DateTime @default(now())
- updatedAt DateTime @updatedAt

### ContentTemplate (new)
- id String @id @default(cuid())
- name String
- type String — newsletter | correspondence | blog
- content String — rich text HTML/MDX
- variables String — JSON array of {key, label, defaultValue}
- createdAt DateTime @default(now())
- updatedAt DateTime @updatedAt

## API Routes

### New Routes
- `POST /api/ai/generate-blog` — Takes { topic, category, keywords }, calls ZAI SDK, returns structured blog post
- `POST /api/ai/auto-schedule` — Generates bi-weekly topics, creates BlogPost records with autoGenerated=true
- `GET/PATCH /api/newsletters` — Newsletter CRUD
- `GET/PATCH /api/newsletters/[id]` — Single newsletter operations
- `GET/PATCH /api/correspondence` — Correspondence CRUD
- `GET/PATCH /api/correspondence/[id]` — Single correspondence operations
- `GET/POST/PATCH/DELETE /api/templates` — Template CRUD
- `GET/PATCH /api/templates/[id]` — Single template operations

### Existing Routes (unchanged)
- `/api/blog-posts` — Blog CRUD
- `/api/blog-posts/[id]` — Single blog operations
- `/api/partners`, `/api/partners/[id]` — Partner management
- `/api/settings` — Site settings
- `/api/admin` — Admin login

## AI Integration

### Provider
z-ai-web-dev-sdk (ZAI) — user has credentials

### Configuration
- `ZAI_API_KEY` and `ZAI_BASE_URL` in .env
- Singleton ZAI client instance in `src/lib/ai.ts`

### Blog Generation Flow
1. Admin provides: topic, category, optional keywords
2. System prompt instructs AI to generate SEO-optimized content
3. AI returns structured JSON: title, metaTitle, metaDescription, excerpt, content (markdown), tags, suggested readTime
4. Admin reviews in rich text editor, edits, saves or publishes

### Auto-Scheduling Flow
1. Admin triggers auto-scheduler
2. System generates ~26 bi-weekly topic ideas for remainder of 2026 (Apr-Dec = ~18 weeks = ~9 posts)
3. For each topic, calls ZAI to generate full article
4. Creates BlogPost records with autoGenerated=true, scheduledAt=bi-weekly dates, published=false
5. Admin can review/edit each before publishing

## Rich Text Editor
- @mdxeditor/editor (already installed, v3.39.1)
- Used for: blog content, newsletter body, correspondence body, template content
- Features: headings, bold/italic, lists, links, images (URL), code blocks

## Content Calendar
- Monthly view with day cells
- Color-coded dots: blog posts (gold), newsletters (blue), correspondence (navy)
- Click date to view/add content
- Filter by content type
- Shows scheduled vs published status

## Template Builder
- Variable injection with live preview
- Standard variables: {{recipient_name}}, {{organization}}, {{date}}, {{campaign_title}}, {{amount}}, {{sender_name}}, {{sender_title}}
- Templates reusable across newsletters and correspondence
- Variables resolved at compose time with real data

## Implementation Constraints
- Keep existing hash-based SPA routing pattern
- Follow existing shadcn/ui component patterns
- All new admin views register under the existing 'admin' page ID with internal view switching
- Use existing Prisma client singleton from src/lib/db.ts
- No new npm packages needed (mdxeditor already installed, z-ai-web-dev-sdk already installed)
